## 1. 서비스 개요

### 1.1 서비스 명 (가칭)
**SummaryTube** (또는 원하는 이름으로 변경)

### 1.2 핵심 가치 제안
- YouTube URL 하나만 넣으면 → **구조화된 요약 + 타임스탬프 + AI 채팅** 제공
- 한국어 완전 지원 (자막 번역, 요약 한국어 출력)
- 영상 외 PDF, 블로그, 오디오 파일도 처리

### 1.3 타겟 사용자
| 세그먼트 | 니즈 |
|---|---|
| 직장인·학습자 | 긴 강의/세미나 영상 빠르게 파악 |
| 콘텐츠 크리에이터 | 경쟁 채널 트렌드 분석 |
| 연구자·학생 | 학술 영상·PDF 핵심 추출 |
| 뉴스 소비자 | 뉴스/다큐 빠른 소화 |

## 2. 경쟁사(lilys.ai) 분석

### 2.1 핵심 기능
```
YouTube URL 입력
    ↓
자막(Transcript) 추출 (YouTube API 또는 Whisper STT)
    ↓
LLM 요약 (Claude / GPT-4o / Gemini 중 선택)
    ↓
구조화된 노트 생성 (타임스탬프 링크 포함)
    ↓
확장 분석: 마인드맵 / 인포그래픽 / 퀴즈 / 플래시카드
    ↓
AI 채팅 (Lily) — 맥락 기반 Q&A
```

### 2.2 lilys.ai 강점
- 100만+ 사용자
- 멀티 포맷 지원 (영상·PDF·블로그·오디오·실시간 녹음)
- Chrome 확장 프로그램으로 원클릭 저장
- SOC 2 보안 인증

### 2.3 공략 가능한 빈틈 (차별화 포인트)
- **한국어 최적화** — 한국어 자막 처리, 한국어 요약 품질 우선
- **더 저렴한 가격** or 더 관대한 무료 티어
- **공유 기능** — 요약 노트 퍼블릭 공유 / 팀 협업
- **채널 구독 알림** — 신규 영상 자동 요약 알림

## 3. 기술 스택

### 3.1 전체 아키텍처
```
[사용자 브라우저]
    ↕ HTTPS
[Next.js 14 App Router — Vercel 배포]
    ↕
[API Routes / Route Handlers]
    ├── YouTube Transcript 추출 모듈
    ├── Whisper STT 모듈 (자막 없는 영상)
    ├── LLM 요약 모듈 (Claude API)
    ├── 파일 처리 모듈 (PDF/오디오)
    └── 인증 모듈
    ↕
[PostgreSQL — Neon Serverless]   [Redis — Upstash]
[파일 스토리지 — Vercel Blob]
```

### 3.2 기술 선택 상세

#### Frontend
| 항목 | 선택 | 이유 |
|---|---|---|
| Framework | **Next.js 14 (App Router)** | SSR/SSG, API Routes 통합, Vercel 최적화 |
| UI | **shadcn/ui + Tailwind CSS** | 빠른 개발, 접근성, 커스터마이징 용이 |
| 상태관리 | **Zustand** | 가볍고 직관적 |
| 마인드맵 | **React Flow** | 노드 기반 시각화 |
| 아이콘 | **Lucide React** | shadcn과 통일 |

#### Backend / API
| 항목 | 선택 | 이유 |
|---|---|---|
| API | **Next.js Route Handlers** | 별도 서버 불필요 |
| LLM | **Claude claude-sonnet-4-6 (Anthropic SDK)** | 긴 컨텍스트, 한국어 우수 |
| STT | **OpenAI Whisper API** | 영상 자막 없을 때 폴백 |
| YouTube Transcript | **youtube-transcript npm** | 자막 직접 추출 |
| 큐 시스템 | **Vercel Queues** (or BullMQ + Redis) | 긴 처리 작업 비동기화 |
| 인증 | **NextAuth.js v5** | Google/Kakao OAuth + 이메일 |

#### 데이터베이스
| 항목 | 선택 |
|---|---|
| 주 DB | **Neon PostgreSQL** (Vercel Marketplace) |
| ORM | **Prisma** |
| 캐시 | **Upstash Redis** |
| 파일 | **Vercel Blob** |

#### 인프라
- **호스팅**: Vercel (Fluid Compute)
- **CDN**: Vercel Edge Network
- **모니터링**: Vercel Analytics + Sentry

## 4. 데이터베이스 스키마

```sql
-- 사용자
users (
  id          UUID PRIMARY KEY,
  email       VARCHAR UNIQUE NOT NULL,
  name        VARCHAR,
  avatar_url  VARCHAR,
  plan        ENUM('free', 'pro', 'team') DEFAULT 'free',
  created_at  TIMESTAMP DEFAULT NOW()
)

-- 요약 노트
notes (
  id              UUID PRIMARY KEY,
  user_id         UUID REFERENCES users(id),
  source_type     ENUM('youtube', 'pdf', 'audio', 'blog', 'text'),
  source_url      VARCHAR,           -- YouTube URL 등
  source_title    VARCHAR,
  source_thumbnail VARCHAR,
  duration_sec    INTEGER,           -- 영상 길이 (초)
  transcript      TEXT,              -- 원본 자막 전체
  summary         JSONB,             -- 구조화된 요약
  language        VARCHAR DEFAULT 'ko',
  is_public       BOOLEAN DEFAULT FALSE,
  status          ENUM('pending','processing','done','error'),
  created_at      TIMESTAMP DEFAULT NOW()
)

-- 자막 세그먼트 (타임스탬프 연동)
transcript_segments (
  id        UUID PRIMARY KEY,
  note_id   UUID REFERENCES notes(id),
  start_sec FLOAT,
  end_sec   FLOAT,
  text      TEXT,
  seq       INTEGER
)

-- 채팅 메시지
chat_messages (
  id         UUID PRIMARY KEY,
  note_id    UUID REFERENCES notes(id),
  user_id    UUID REFERENCES users(id),
  role       ENUM('user', 'assistant'),
  content    TEXT,
  created_at TIMESTAMP DEFAULT NOW()
)

-- 컬렉션 (폴더)
collections (
  id      UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  name    VARCHAR,
  icon    VARCHAR
)

collection_notes (
  collection_id UUID REFERENCES collections(id),
  note_id       UUID REFERENCES notes(id),
  PRIMARY KEY (collection_id, note_id)
)

-- 하이라이트
highlights (
  id         UUID PRIMARY KEY,
  note_id    UUID REFERENCES notes(id),
  user_id    UUID REFERENCES users(id),
  text       TEXT,
  start_sec  FLOAT,
  color      VARCHAR DEFAULT 'yellow',
  created_at TIMESTAMP DEFAULT NOW()
)

-- 사용량 추적 (플랜 제한)
usage_logs (
  id         UUID PRIMARY KEY,
  user_id    UUID REFERENCES users(id),
  action     VARCHAR,  -- 'summarize', 'chat', 'export'
  tokens     INTEGER,
  created_at TIMESTAMP DEFAULT NOW()
)
```

## 5. API 설계

### 5.1 핵심 엔드포인트

```
POST   /api/notes                    # 새 요약 생성 (YouTube URL 제출)
GET    /api/notes                    # 내 노트 목록
GET    /api/notes/:id                # 노트 상세 조회
DELETE /api/notes/:id                # 노트 삭제
PATCH  /api/notes/:id                # 노트 수정 (제목, 공개 여부)

GET    /api/notes/:id/status         # 처리 상태 폴링 (SSE)

POST   /api/notes/:id/chat           # AI 채팅 (스트리밍)
GET    /api/notes/:id/chat           # 채팅 히스토리

POST   /api/notes/:id/highlights     # 하이라이트 추가
DELETE /api/notes/:id/highlights/:hid

POST   /api/notes/:id/export         # 내보내기 (Markdown/PDF)

GET    /api/collections              # 컬렉션 목록
POST   /api/collections              # 컬렉션 생성
POST   /api/collections/:id/notes    # 노트를 컬렉션에 추가

GET    /api/user/usage               # 사용량 조회
GET    /api/user/plan                # 플랜 정보
```

### 5.2 핵심 처리 흐름 (POST /api/notes)

```
1. YouTube URL 유효성 검증
2. 이미 처리된 URL인지 캐시 확인 (Redis)
3. DB에 note 레코드 생성 (status: 'pending')
4. 백그라운드 Job 큐에 추가
5. note_id 즉시 반환

--- 백그라운드 Job ---
6. YouTube 영상 메타데이터 조회 (제목, 썸네일, 길이)
7. 자막(Transcript) 추출 시도
   ├── youtube-transcript 라이브러리로 자막 추출
   └── 실패 시 → Whisper API로 오디오 STT
8. transcript_segments DB 저장
9. Claude API로 요약 생성 (프롬프트 엔지니어링)
10. summary JSONB 저장 (status: 'done')
11. 사용자에게 완료 알림 (SSE or Webhook)
```

## 11. 성능 최적화 전략

1. **자막 캐싱**: 동일 URL 재요청 시 Redis에서 즉시 반환
2. **스트리밍 응답**: Claude API 스트리밍으로 요약 실시간 표시
3. **비동기 처리**: 긴 영상(1시간+)은 큐 기반 백그라운드 처리
4. **청크 분할**: 자막이 너무 길면 섹션별 분할 요약 후 통합
5. **ISR**: 공개 노트 페이지는 Incremental Static Regeneration 적용

## 14. 개발 시작 순서 (Week by Week)

## 15. 참고 라이브러리 & 리소스

|---|---|
| YouTube 자막 | `youtube-transcript` |

| Claude LLM | `@anthropic-ai/sdk` |
| OpenAI Whisper | `openai` |
| ORM | `prisma` |
| 인증 | `next-auth` |
| 마인드맵 | `reactflow` |
| PDF 처리 | `pdf-parse` |
| Markdown | `react-markdown` + `remark-gfm` |
| 내보내기 | `jspdf` + `html2canvas` |
| 드래그앤드롭 | `@dnd-kit/core` |
| 날짜 | `date-fns` |
